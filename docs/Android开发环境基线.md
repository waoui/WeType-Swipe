## 每次升级前检查

- [ ] 已导出当前模块配置。
- [ ] 已备份当前可用 APK。
- [ ] 已记录 Root、LSPosed、微信输入法和系统版本。
- [ ] 已确认至少保留一台不升级的稳定测试机。
- [ ] 已准备恢复方式或完整系统备份。
- [ ] 已确认升级不是为了解决与当前项目无关的问题。

## 维护规则

1. 稳定设备不追新版本，只在明确收益或兼容需求出现时升级。
2. 每次只改变一个关键变量，例如先升级微信输入法，不同时升级系统和 LSPosed。
3. 新版通过完整回归测试后，才能更新“主测试机”基线。
4. 发现问题时优先比较升级前后的环境差异，再修改 Hook 代码。
