# AGENTS.md

## 项目定位

本仓库是安卓版微信输入法下滑快捷键 LSPosed 模块。目标是保证功能稳定、兼容清晰、发布流程可重复，不为追求新功能破坏既有用户配置和已验证环境。

## 开发原则

1. 优先修复兼容性、崩溃、配置丢失和交互问题，再考虑新增功能。
2. 不修改现有功能语义、默认键位和配置存储格式，除非任务明确要求；涉及迁移时必须提供向后兼容逻辑。
3. 微信输入法内部类名、资源名和 View 层级可能随版本变化。新增 Hook 必须：
   - 先使用稳定特征匹配；
   - 保留失败日志；
   - 避免单点硬编码；
   - 不得因匹配失败导致微信输入法崩溃。
4. 模块不得联网，不得采集、上传或保存用户输入内容。
5. 不提交签名文件、密钥、账号、Token、真实设备日志中的隐私信息或本地绝对路径。
6. 修改 UI 时必须同时检查：状态栏遮挡、深浅色模式、26 键布局、九宫格布局、横屏、悬浮键盘和小屏设备。
7. 修改手势逻辑时必须避免影响正常点击、长按、滑行输入和原生手势。

## 兼容基线

- Android：8.0 及以上
- 当前重点测试：Android 16
- 构建：JDK 17、Android SDK 35、Gradle 8.9
- 运行环境：LSPosed
- 当前重点兼容的微信输入法版本以 README 和 CHANGELOG 为准

## 必须执行的验证

提交前执行：

```bash
./gradlew clean testDebugUnitTest lintDebug assembleDebug
```

Windows：

```powershell
gradlew.bat clean testDebugUnitTest lintDebug assembleDebug
```

如任务修改 Hook、反射、资源匹配、配置序列化或发布流程，还必须补充对应单元测试或明确记录无法自动化验证的原因，并给出真机验收步骤。

## 版本与发布

- 版本号只在 `gradle.properties` 的 `VERSION_CODE`、`VERSION_NAME` 中维护。
- 正式发布必须同步更新：
  - `CHANGELOG.md`
  - `RELEASE_NOTES_v版本号.md`
  - `README.md` 中的稳定版本与兼容信息
- 不在本地提交正式签名产物。正式 Release 使用 GitHub Actions 和仓库 Secrets。
- 不跳过 CI，不直接修改已发布 Tag。

## 分支与提交

- 日常任务从 `main` 创建 `codex/<任务名>` 分支。
- 每个分支只处理一个明确任务，避免把无关重构混入修复。
- 提交信息使用中文，说明实际变化，例如：
  - `修复微信输入法 3.5.2 剪贴板入口匹配`
  - `增加九宫格布局回归测试`
- 完成后创建 PR，写明：问题、修改、测试结果、真机验收项和回滚方式。

## 禁止事项

- 禁止为通过测试而删除有效测试或弱化断言。
- 禁止无依据升级 Gradle、AGP、Kotlin、Xposed API 或最低 Android 版本。
- 禁止大范围重写已稳定 Hook，除非现有实现无法维护且任务明确要求。
- 禁止把调试日志默认长期保持在高频输出状态。
